# Implementation Plan - Busy Dad Burpee Timer

## 1. Technical Context
- **Framework:** React Native (Expo) or Flutter (assuming React/Web for this spec context, but adaptable). *Decision: React (Web/PWA) for immediate prototyping.*
- **State Management:** React `useState` + `useEffect` for timer loop.
- **Precision:** `requestAnimationFrame` or `performance.now()` for metronome smoothness, correcting drift against a start timestamp.
- **Audio:** `HTML5 Audio` or `Howler.js` for low-latency beeps.

## 2. Project Structure
- `src/components/ConfigurationForm.tsx`: Inputs for Reps, Sets, Ratio.
- `src/components/WorkoutSession.tsx`: Main timer logic and active view.
- `src/components/MetronomeBar.tsx`: Visual component for the Up/Down animation.
- `src/utils/calculator.ts`: Core math for Pacing and Rest.
- `src/hooks/usePreciseTimer.ts`: Custom hook to handle the 20-minute loop without drift.

## 3. Complexity Estimates
- **Calculator Logic:** Low. Simple arithmetic.
- **Metronome Animation:** Medium. Ensuring smooth 60fps filling synced exactly to `TimePerRep` requires precise CSS transitions or Canvas drawing.
- **Audio Sync:** Low. Standard browser APIs.
